üì• Installation

Quick Install (Recommended)

curl -sL https://raw.githubusercontent.com/stanzinofree/kcsi/main/install.sh | bash

Manual Installation

1
Download the latest release
wget https://github.com/stanzinofree/kcsi/releases/latest/download/kcsi-linux-amd64
2
Make it executable and move to PATH
chmod +x kcsi-linux-amd64
sudo mv kcsi-linux-amd64 /usr/local/bin/kcsi
3
Set up autocompletion
echo 'source <(kcsi completion bash)' >> ~/.bashrc
source ~/.bashrc
üí° Tip: For Zsh users, replace bash with zsh and use ~/.zshrc

üöÄ Quick Start Guide

Basic Commands

Command Description
kcsi get pods -n <TAB> List pods with namespace autocomplete
kcsi logs -f -n <ns> <pod> Follow pod logs with cascading autocomplete
kcsi describe pod -n <ns> <pod> Describe a pod with autocomplete
kcsi delete pod -n <ns> <pod> Delete pod with confirmation prompt
kcsi events -w Watch cluster events in real-time
kcsi check errors Find all problematic pods

Resource Shortcuts

Full Command Shortcut

get deployments get deploy
get namespaces get ns
get configmaps get cm
get nodes get no

üí° Practical Examples

Example 1: Debugging a Crashing Pod

kcsi check errors

Find all pods with issues

kcsi logs -f -n production api-server-xxx

Follow the logs of the problematic pod

kcsi describe pod -n production api-server-xxx

Get detailed information about the pod

Example 2: Monitoring Production

kcsi events -n production -w

Watch events in the production namespace

kcsi get pods -n production -o wide

See which nodes the pods are running on

Example 3: Multi-Container Pod Logs

kcsi logs -n default my-pod -c <TAB>

Press TAB to autocomplete container names

kcsi logs -f -n default my-pod -c sidecar --tail 100

Follow last 100 lines from specific container

üí° Pro Tip: Use -o yaml or -o json flags with any get command to export configurations for backup or migration!

üéì Advanced Features

Safety Mechanisms

All delete operations require confirmation by default:

kcsi delete pod -n production my-pod

‚ö† Are you sure you want to delete pod 'my-pod' in namespace 'production'? [y/N]:

Skip confirmation for automation:

kcsi delete pod -n production my-pod --force

Output Formats

Format Use Case Example
-o wide See node placement and IPs kcsi get pods -o wide
-o yaml Export/backup configurations kcsi get deploy -o yaml
-o json Programmatic processing kcsi get svc -o json

Diagnostics Tools

Check for pod errors:

kcsi check errors

Shows all pods not in Running or Completed state with helpful suggestions

Monitor cluster events:

kcsi events -n <namespace> -w

Watch events in real-time, sorted by timestamp
